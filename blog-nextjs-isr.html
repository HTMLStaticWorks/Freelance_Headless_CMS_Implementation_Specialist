Incremental Static Regeneration.

Leveraging ISR in Next.js to handle real-time content updates from your Headless CMS without full rebuilds.

Static Site Generation (SSG) is great for performance, but it can be a nightmare for sites with thousands of pages or frequent content updates. This is where ISR comes in.

What is ISR?

ISR allows you to create or update static pages *after* you’ve built your site. You can use static-generation on a per-page basis, without needing to rebuild the entire site.

// Next.js ISR Example
export async function getStaticProps() {
  const res = await fetch('https://api.cms.com/posts')
  const posts = await res.json()

  return {
    props: {
      posts,
    },
    // Revalidate every 60 seconds
    revalidate: 60,
  }
}

On-Demand Revalidation

With the latest versions of Next.js, you can even trigger revalidation manually via a webhook from your CMS. This means as soon as a writer clicks "Publish", your site updates instantly.

Conclusion

ISR provides the best of both worlds: the performance of static sites and the flexibility of dynamic ones. It's a game-changer for Headless CMS architectures.
